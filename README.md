# Animal health care

# 概要  
このアプリケーションは飼い主によりペットの様子を記録できます。毎日記録する手間を最小限にするために記述式ではなく,質問形式で選択肢から回答を選び記録します。  
記録することでペットの体調の変化を可視化し、飼い主がペットの体調を管理しやすくなります。また、ペットの情報を獣医師へ共有する際に、記録された内容から正確な情報を共有できます。つまり、日々の記録がペットの状態を判断する情報源となり、予防あるいは対処療法へ役立つアプリケーションとなっております。

# 制作背景（意図）

飼い主が獣医師へペットの情報を共有する際、獣医師が必要とする情報を正確に伝えることで、ペットの状態をより正確な判断ができるのではないかと考えました。そこで、ポイントを絞って日々ペットの状態を記録し、曖昧な情報ではなく正確な情報を伝達することを目的に開発に挑みました。

また、動物に関する知識不足が原因で動物の体調管理を誤り、事故に繋がることを危惧していました。そこで、普段何に気をつけるべきか観察ポイントを絞って日々ペットの状態を記録することで、ペットと過ごす上で必要なポイントを抑えられるようなアプリケーションを開発しようと考えました。

# 工夫したポイント  
ユーザーが継続してアプリケーションを利用できるように、カレンダーを導入しました。人は心理的に貯め続けたものを止めることに抵抗を感じると知り、トップページにカレンダーを表示し、記録した日を色付け視覚的な効果から継続しやすい状態を作ろうと考えました。なぜなら、このアプリケーションは記録し続けてこそ効果を発揮する為です。また、記録する時に記述形式だと手間が増えてしまい、継続しづらい状況になると考えました。そこで、質問形式かつ回答を選択式にすることで、手間を最小限に減らす工夫を凝らしました。

# 洗い出した要件

* ユーザー管理機能  
目的：ユーザー情報の登録／ログイン／ログアウトをできるようにする為です。  
詳細：新規登録ボタンをクリックしユーザー情報の登録すること、また登録済みのユーザーはログイン/ログアウトのボタンからアプリ利用が可能です。  
ユースケース：ユーザーアカウントの登録ボタンをクリックすると、登録情報が表示され必須項目を入力します。入力後に登録するボタンをクリックし登録内容はデータベースに保存されます。

* 投稿機能  
目的：ユーザーがペットの情報を投稿できるようにする為です。  
詳細：記録するボタンをクリックして動物の様子を７つの質問に対し５つの選択肢から該当するものを１つ選び、必要に応じてメモ欄へ入力、または画像を挿入し記録するボタンをクリックすると投稿できます。  
ユースケース：・ユーザーが記録するボタンをクリックすると、動物の体調について７つの質問に対して５つの選択肢から１つクリックし回答します。いつもと違う日は、内容をメモ欄に入力し、必要であれば画像を貼り付けます。必要な情報の入力を終えたら記録するボタンをクリックして投稿します。

* 投稿一覧表示機能  
目的：ユーザーが投稿の一覧を確認できるようにする為です。  
詳細：トップページのカレンダーに質問回答が表示され投稿の一覧を確認できます。  
ユースケース：トップページのカレンダーに質問と選択された回答が表示され投稿を一覧表示します。いつもと違う点がある場合はカレンダーの対象日に＊マークが表示されます。記録された内容を元にユーザーが動物の体調管理を行うこと、また獣医師と情報共有する際に使用することが可能です。

* 投稿詳細表示機能  
目的：ユーザーが投稿の詳細を確認できるようにする為です。  
詳細：詳細ボタンをクリックして投稿した内容の詳細が表示されます。  
ユースケース：ユーザーが詳細ボタンをクリックすると、投稿された内容の詳細を確認できます。詳細ページには編集ボタン、削除ボタン、選択した日付、質問回答、メモ内容、画像が表示されます。ユーザーが左上のアプリ名をクリックすると、トップページへ遷移します。

* 投稿編集機能  
目的：ユーザーが投稿の編集を行えるようにする為です。  
詳細：編集ボタンをクリックして投稿した内容の編集が行えます。  
ユースケース：ユーザーが投稿の詳細ページを開き、編集ボタンをクリックすると、選択された投稿内容の編集画面へ遷移されます。ユーザーが投稿内容を編集した後、変更するボタンをクリックすると詳細ページへ遷移します。

* 投稿削除機能  
目的：ユーザーが投稿の削除を行えるようにする為です。  
詳細：削除ボタンをクリックして投稿した内容を削除できます。  
ユースケース：ユーザーが投稿の詳細ページを開き、削除ボタンをクリックすると選択された投稿内容を削除できます。ユーザーが削除ボタンをクリックすると投稿が削除されます。

# 課題や今後実装したい機能

* 課題  
開発全体を通して、開発の段取りが悪かったと感じております。なぜなら、１つの機能の実装中に他の機能も実装してしまい、実装の漏れが発生し完成までに時間がかかってしまった為です。そこで、気になった点はメモをとり、実装が完了した後にメモを確認し、必要であれば実装するよう工夫しました。

* 今後実装したい機能１  
質問回答のグラフ化です。質問回答をグラフ化することで、ユーザーがペットの記録から変化をより分かりやすく読み取る為に実装したいと考えております。

* 今後実装したい機能２  
１つのユーザーで複数のペットの体調管理を可能にすることです。現時点では１つのユーザーでペット１匹の情報しか記録できないため、多頭飼いされているユーザー向きではありません。より多くの方が利用できるアプリケーションにするために必要だと考えております。
